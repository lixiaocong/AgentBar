## 目标

把 reset 从「写入时分类记录」改为「展示时自动检测 + 竖线展示」。存储层不再记录 reset，只在图表渲染时检测 remaining% 突然跳升到接近满额，画一条竖线（`RuleMark`）标记。

检测规则（按你确认的 ≥95% 且有提升）：扫描连续样本，当某样本 `remaining ≥ 95%` **且** 上一个样本 `remaining < 95%` 时，判定为一次 reset。这样一次 reset 只产一条竖线，不会在接近满额的平段刷标记；持续接近满额的账号也不会误标。

## 改动清单

### 1. `Sources/AgentBar/History/QuotaHistoryModels.swift`

- **`QuotaHistoryEventKind`**：删除 `.reset` 和 `.likelyReset` 两个 case。保留 `initial / interval / changed / scheduleChanged`（schedule 变化检测不在本次范围内，保留）。
- **`title`**：删除 `.reset` / `.likelyReset` 分支，只剩 `.scheduleChanged` 返回标题。
- **删除 `isResetMarker`** 计算属性。
- **`QuotaHistoryDownsampler.downsample`**：删除 `candidates.append(contentsOf: bucket.filter { $0.eventKind.isResetMarker })` 这一行（枚举已无该属性）。
- **新增纯函数检测器**（可单测，无需 UI/DB）：
  ```swift
  enum QuotaHistoryResetDetector {
      static let thresholdPercent: Double = 95
      static func resetDates(in samples: [QuotaHistorySample]) -> [Date]
  }
  ```
  遍历非 unlimited 样本，按上述"跨入 ≥95% 区间"规则收集 reset 时间点。

### 2. `Sources/AgentBar/History/QuotaHistoryStore.swift`

- **`eventKind(...)` 静态函数**：删除整段 reset 分类逻辑（`balanceIncrease`、`previousResetPassed`、`scheduleAdvanced`、`.reset`/`.likelyReset` 分支）。简化为：
  - `resetChanged` → `.scheduleChanged`
  - `amountChanged || labelsChanged || unlimitedChanged` → `.changed`
  - `intervalElapsed` → `.interval`
  - 否则 `.changed`
- 删除 `likelyResetBasisPoints` 常量（仅 reset 分类用）。**保留** `resetScheduleToleranceMilliseconds` 和 `resetScheduleChanged`（仍服务于 schedule 变化检测和去重）。
- **保留去重逻辑**（`amountChanged/labelsChanged/resetChanged/unlimitedChanged/intervalElapsed`）——这点关键：reset 跳升仍会作为 `.changed` 样本被记录，我们正是靠这些数据点在展示时检测跳变。
- 精简 `eventKind` 的参数列表（去掉不再使用的入参）。
- schema **不动**：`event_kind` 列保持 NOT NULL，写入仍绑定合法 raw 值。旧库里历史的 `.reset`(3)/`.likelyReset`(4) 读取时 `init(rawValue:)` 返回 nil → 回退 `.changed`，无数据丢失。

### 3. `Sources/AgentBar/Views/QuotaHistoryView.swift`

- **删除** `resetSamples` 计算属性和 `ForEach(resetSamples) { PointMark ... }` 块（原圆点标记）。
- **新增** 用 `QuotaHistoryResetDetector.resetDates(in: numericSamples)` 得到 reset 时间点，渲染为竖线：
  ```swift
  ForEach(detectedResetDates, id: \.self) { date in
      RuleMark(x: .value("Reset", date))
          .foregroundStyle(.green.opacity(0.6))
          .lineStyle(StrokeStyle(lineWidth: 1.5, dash: [4, 3]))
          .annotation(position: .top) {
              Image(systemName: "arrow.clockwise").font(.caption2).foregroundStyle(.green)
          }
  }
  ```
- **`historySymbolName` / `historyTint` 扩展**：删除 `.reset` / `.likelyReset` 分支（枚举已无这些 case，编译器会要求穷尽性更新）。
- 样本详情行（`sampleDetail`）保留，现在只会显示 `.scheduleChanged` 标签或空。

### 4. `Tests/AgentBarTests/QuotaHistoryStoreTests.swift`

- **`quotaHistoryDistinguishesConfirmedLikelyAndScheduleEvents`**：重写。原断言 `.reset`/`.likelyReset` 已不成立。改为断言：balance 跳升现在记录为 `.changed`/`.scheduleChanged`，reset 不再被存储分类。
- **`quotaHistoryDownsamplingPreservesResetMarkers`**：重写。原测试依赖 `.reset` eventKind。改为验证降采样保留首/尾/极值点（去掉 reset 专项断言）。
- **新增** `quotaHistoryResetDetectorMarksJumpsAboveThreshold`：纯函数测试 `QuotaHistoryResetDetector`——构造 remaining 序列 `20→22→18→98→50→97`，断言在 98 和 97 处各检出一次 reset；持续 ≥95% 的平段不重复标记。
- 其余测试（`...InitialChangesAndFifteenMinuteHeartbeat`、`...RollingResetCountdownAsUnchanged`、`...UnlimitedAndDeletesByCutoff`）的断言不受影响，保持不变。

### 5. 文档（`AGENTS.md`、`README.md`）

- AGENTS.md：删除"Mark a balance increase as Reset/Likely reset"那条 history rule；更新测试表里被重命名的测试描述。
